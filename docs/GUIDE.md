## AI 写作提示：高质量技术博客

1. **主题聚焦 + 明确读者收益**  
   - 开头一句话说明要解决的问题或要学习的技能，并点出读者完成文章后能够掌握什么。  
   - 写作前列出 1 句“读者痛点 + 解决方案”，确保整篇文章都围绕它展开。

2. **结构清爽、层级统一**  
   - 采用 `前言 -> 背景/问题 -> 方案/实战 -> 总结/延伸` 的骨架，标题从 H2 开始，段落不超过 4 句。  
   - 每个代码块、图示都要有小标题和 1 句上下文说明，避免读者猜含义。

3. **实战细节优先于空泛概念**  
   - 每个要点至少给出一个真实场景（复现步骤、踩坑记录、性能对比等）。  
   - 说明“问题如何暴露、如何定位、尝试过哪些方案、最终为何选择当前方案”。

4. **代码 / 命令可复制即可运行**  
   - 说明必要的环境：系统、Node/包版本、依赖、配置片段。  
   - 每段代码都提供输入 → 输出或预期效果，必要时补充验证命令。

5. **观点清晰 + 作者思考**  
   - 至少包含一段“我为什么这样做/换个场景如何取舍”的分析，突出判断依据或权衡。  
   - 如果存在局限或替代方案，明确告知并给出参考链接/关键字。

6. **恰当的图示与对比**  
   - 复杂流程优先用表格、列表或流程图描述；若无法提供图，也要用分步文字模拟。  
   - 图文紧邻引用处，并提供文字说明避免无障碍问题。

7. **保持紧凑，剔除冗余**  
   - 删除与主题无关的背景故事，直接给结论、原因、操作步骤。  
   - 复查每段：若去掉不影响理解，就删。

8. **SEO / 可发现性（可选）**  
   - 标题包含核心关键词（如“Safari 100vh 问题”），正文自然重复 2-3 次相关词。  
   - 在总结或 FAQ 中补充“搜索常见问法 + 回答”，提升被检索到的机会。

> 写作流程示例：**明确受众 → 列大纲 → 填入踩坑 & 代码 → 加结论/复盘 → 自检结构/SEO**。  
> 只有当所有步骤都覆盖上述 8 条时，才算完成高质量技术博客。***
